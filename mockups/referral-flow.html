Referral mechanic — visual flow

Як працює: signup → nanoid(8) ref_code → user shares → friend signs з ?ref= → invite_count++ → ≥3 = priority access email. Без backend, тільки n8n + Airtable.

Notes

  • V1 Actor sequence — для presentation product flow комусь нетехнічному.
  • V2 Data flow — для тебе самого як reference під час n8n implementation.
  • V3 Comic story — для blog post чи Telegram-каналу що пояснює users як це працює.
  • Threshold у ради: 3 invites = priority. K-factor target ≥ 0.25 = ~25% юзерів запросять хоча б 1 друга. Якщо K-factor < 0.3 за 30 днів → switch до position-bump mechanic.
  • Без backend — все через n8n + Airtable + URL params. Жодного JWT, жодних cookies.
