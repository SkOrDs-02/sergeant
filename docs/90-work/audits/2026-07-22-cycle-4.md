## 2 · `page-enter` опт-аут — виміряно, критерій закрито

Вирок «one entry system per page» виконано для `/pricing`.

|                         | До                          | Після          |
| ----------------------- | --------------------------- | -------------- |
| running CSSAnimation    | **3**                       | **2**          |
| running names           | `page-enter`, `fadeSlideUp` | `fadeSlideUp`  |
| RESPONSE-груп одночасно | **2** (обгортка + стагер)   | **1** (стагер) |
| inline delays           | `0ms` / `30ms`              | `0ms` / `30ms` |

Обидва виміри — prod-збірка, rAF-полінг до появи `[style*='animation-delay']`.

**Реалізація без прапорця, свідомо.** Наряд просив «прапорець опт-ауту»,
але обгортка `page-enter` і так пишеться **окремо в кожному записі**
реєстру — там немає спільного місця, з якого треба «виписуватись». Прапорець
додав би шар конфігурації поверх шести інлайнових `<div className="page-enter">`,
не змінивши нічого, крім кількості місць, які треба тримати в голові.
Опт-аут = не писати обгортку, і саме так `/welcome` жив уже до цього циклу.
Правило зафіксовано текстом у Hard Rule #17 — воно і є механізмом.

**Аудит решти standalone-маршрутів** (grep по `animation-delay` / `stagger`):
власну вхідну хореографію мають лише `/pricing` і `/welcome`. Решта
(`/sign-in`, `/reset-password`, legal, `/status`, `/assistant`) внутрішніх
стагерів не має — `page-enter` для них лишається правильним дефолтом.

**Побічно знайдено й полагоджено два порушення кроку стагеру** (Hard Rule
#17 вимагає ≤30 мс між дітьми):

| Місце                               | Було                              | Стало                                 |
| ----------------------------------- | --------------------------------- | ------------------------------------- |
| `WelcomeOneScreen` — картки модулів | `idx * 50` мс                     | `Math.min(idx * 30, 150)`             |
| `WelcomeScreen` — peek-картки       | `0.4 + idx * 0.1` с (крок 100 мс) | `calc(0.4s + Math.min(idx*30,150)ms)` |

Базова затримка `0.4s` у peek-групі лишена: це затримка входу **всієї
групи**, а не крок між дітьми.

**Відкрите:** `/welcome` тримає дві стагер-групи (картки модулів +
peek-картки) плюс три ambient-глоу одночасно. За буквою правила це знову
більше ніж «1 AMBIENT + 1 RESPONSE». Не чіпав — це питання композиції
splash-екрана, а не таймінгу, і воно за тобою.

## 3 · StreakFlame → тем-залежні токени

Кольори переїхали з гілок компонента в токени
`--c-streak-tier-{3,7,14,30,60,100}` з парами light/dark. Компонент тепер
лише мапить серію на щабель.

`[c4-streak-ladder--light.png]`, `[c4-streak-ladder--dark.png]` → піпетка по
фактичному рендеру, кожен щабель на своєму тлі:

| Серія | Light     | Контраст на кремі | Dark      | Контраст на ink |
| ----- | --------- | ----------------- | --------- | --------------- |
| 3     | coral-600 | **3.22:1**        | coral-300 | 10.88:1         |
| 7     | coral-600 | **3.22:1**        | coral-400 | 8.18:1          |
| 14    | coral-700 | 4.5:1             | coral-500 | 6.65:1          |
| 30    | coral-700 | 4.5:1             | coral-600 | 4.88:1          |
| 60    | coral-800 | 5.87:1            | coral-700 | **3.49:1**      |
| 100+  | amber-700 | 4.27:1            | amber-400 | 11.1:1          |

**Усі дванадцять щаблів ≥3:1 на своєму тлі.** Борг циклу 3 закрито числами.

**Світлі щаблі зсунуті вниз проти пропозиції наряду.** Наряд задавав старт
зі `coral-500`; піпетка дала для нього **2.80:1 на білому** — під порогом.
Наряд це передбачав («якщо піпетка покаже <3:1 — зсунути щабель, число в
notes»), тож драбина стартує з `coral-600`. Шість серій мапляться на чотири
видимі щаблі: 3/7 і 14/30 діляться попарно, як і планувалось.

**Сота доба отримала власний щабель, а не `--c-celebration`.** Наряд просив
перевикористати наявний токен, але amber-400 дає **1.66:1 на білому**. Для
світлої теми взято AA-компаньйон amber-700 (`warning-strong`, 4.27:1 на
кремі), для темної лишився amber-400. Пара зареєстрована в
`packages/design-tokens/tokens.js` як `celebrationColors` — до цього
святковий hue існував лише сирим значенням у `theme.css`.

**Контракт-тест розширено** — `streakTiers.contract.test.js`: кожен щабель
обох тем мусить бути тиром бренд-палітри, статус- або celebration-токеном;
плюс перевірка монотонності драбини всередині теми. Тест читає `theme.css`
як текст — це єдине джерело правди для тем-залежних пар, і парсити його
дешевше, ніж дублювати значення в JS.

Причина, чому це стало можливим, — саме перенесення в токени: у файлі
токенів шейп стабільний, а гілки функції статично не вирішуються. Це закриває
і те заперечення, з яким контракт-тест не розширили в циклі 3.
