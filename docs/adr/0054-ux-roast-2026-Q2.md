## Context and Problem Statement

Прожарка 2026-05-06 виявила 17 add (A1–A17), 16 change (C1–C16) та 6 remove (R1–R6) пунктів розподілених по 15 секціях (§1 IA, §2 Copy, §3 Empty-states, §4 Auth UX, §5 Forms, §6 Accessibility, §7 Perf, §8 Settings, §9 Experimental, §10 Profile, §11 Privacy, §12 Mobile, §13 Animation, §14 master-list, §15 Pricing). Потрібно зафіксувати ключові архітектурні рішення, прийняті в ході виконання плану, щоб майбутні агенти не регресували їх мимоволі.

## Considered Options

Для кожної з ключових ділянок нижче розглядались очевидні альтернативи (зазначено там, де вибір нетривіальний).

## Decision

### §1 — IA: уніфікація landing-вкладки

Перша вкладка всіх 4 модулів — лейбл **«Огляд»**; `id`-атрибут маршруту залишається незмінним (`today`, `calendar` тощо) для зворотної сумісності email deep-link-ів. ✅ Реалізовано у PR-4 (#2103).

### §2 — Copy: tone-of-voice

- Усі loading-state рядки → **1-а особа однини** («Зберігаю…», «Виходжу…»).
- Zod-валідація → конкретні «Введи X» / «Обери X», без generic `fieldRequired`.
- Реєстраційний success: `«Готово, ${name}!»` — особисте, не корпоративне.
- Документується у `docs/copy/style-guide.uk.md` (PR-X3).

### §3 — Empty-states

- Усі empty-state компоненти → `<EmptyState>` з `compact` / module-tinted variant.
- FTUX bento-картки незаповнених модулів — окремий `<ModuleBentoEmpty>` компонент (PR-20), не inline logic у `HubDashboard`.
- Chat empty → chip-suggestions через `<ChatEmpty>` (PR-26, ✅ done).

### §4 — Auth UX

- Forgot-password панель: auto-collapse через 6 с + кнопка «Назад до входу» (PR-27, ✅ done).
- `ModuleErrorBoundary` — request-ID + copy-button через shared util `core/observability/requestId.ts` (PR-14), не дубляцією `ErrorBoundary.tsx`.

### §5 — Forms

- Password strength → entropy-метрика (char-class × uniqueRatio × len); three levels у `shared/lib/auth/passwordStrength.ts` (PR-15, ✅ done).
- Reminder-time input → єдиний `<TimeInput>` shared-компонент (PR-9).
- Mono error → розрізняємо `401` (токен) vs network-fail у `FinykLoginScreen` (PR-32).

### §6 — Accessibility

- dnd-kit реorder: `KeyboardSensor` + stрілки + aria-live announcement (PR-12).
- Delta ▲▼ для colorblind: icon `trending-up|down` поряд з відсотком (PR-29, ✅ done).
- `OfflineBanner` aria-live: `role="status" aria-live="off"` + `ScreenReaderAnnouncer` (PR-19).

### §7 — Performance

- Lazy-split: `<Suspense>` на рівні sub-cards усередині вже-mount-нутих сторінок (`HubDashboard` Insights/Digest, finyk Overview sub-cards) — **не** дублює route-level lazy з initiative 0006.
- IntersectionObserver як trigger для below-fold секцій.
- PostHog і Sentry ініціалізуються через `dynamic import` після `window.load` (PR-25).

### §8 — Settings IA

- Нова секція «Приватність» (`PrivacySection`) → 4 рядки статусу permissions (PR-7).
- `MemoryBankSection` переїжджає в Settings → Асистент (PR-16).
- Logout primary → `ProfilePage` (PR-17, ✅ done); Settings → General → без logout.

### §9 — Experimental

- Experimental section: верхній banner «можуть бути нестабільними» + confirmation-checkbox при першому ввімкненні (PR-36).
- «Діагностика SW» → переїжджає з основного UI в Experimental (PR-35).

### §11 — Privacy (App Lock)

- App Lock: PIN hash через `SubtleCrypto PBKDF2` у IndexedDB (не localStorage).
- Feature-flag `app-lock-enabled` → default off → Experimental → on-by-default після стабілізації.
- Biometric → opt-in поверх PIN (PR-1b), не заміна; через `BiometricAdapter` unified interface.

### §12 — Mobile

- `100dvh` → `min-h-svh`, `pb-[env(safe-area-inset-bottom)]` на HubMainContent + модульних roots (PR-24).
- `HubBottomNav` fixed-bottom: `bottom: env(safe-area-inset-bottom)`.

### §13 — Animation

- Bento stagger: 0/80/160 ms → 0/30/60 ms (PR-38, ✅ done).
- Bar-grow cap: 600 ms максимум на останній бар (PR-39, ✅ done).

### §15 — Pricing / Chat Counter

- Free tier = 5 AI req/day; Pro = unlimited. **2-tier модель** (ADR-0051 — без Plus).
- Server-side: `effectiveLimits.ts` + `requirePlan.ts` middleware реалізовані у ADR-0051 commit (`30584b1`).
- Frontend counter UI залишається за PR-42.

## Rationale

- **PIN у IndexedDB (не localStorage):** localStorage доступний cross-origin через інʼєкцію; IndexedDB + SubtleCrypto — окремий storage origin, відповідає OWASP MSTG L2.
- **Biometric opt-in, не default:** WebAuthn UX непослідовний між браузерами; iOS PWA без Capacitor-shell не має доступу до Face ID. Opt-in запобігає негативному досвіду при недоступній функції.
- **Sub-card lazy vs route lazy:** route-level split (initiative 0006) знижує initial bundle; sub-card split знижує LCP всередині вже-завантаженої сторінки. Обидва потрібні, конфліктів немає.

## Consequences

### Positive

- Консистентний tone-of-voice у 30+ компонентах.
- App-lock знижує ризик несанкціонованого доступу до фінансових даних.
- LCP ≥ 100 ms покращення після PR-22 + PR-25.

### Negative

- SubtleCrypto iOS Safari < 14 → потрібен polyfill або bail-out banner (документується у `docs/security/app-lock.md`).
- PR-42 залежить від PR-0 — без telemetry бекенду ліміти не вимірюються на фронті.

### Neutral

- PR-13 (sync-error toast) і PR-37 (dark-mode дубль) закриті як obsolete-by-drift — codebase еволюціонував без цих завдань.

## Compliance

- ESLint rule `sergeant-design/no-english-toast-string` (PR-X1) — захищає від регресії EN-strings.
- `docs/copy/style-guide.uk.md` (PR-X3) — tone-of-voice reference для рев'юерів.
- Усі PR мають RTL/Vitest тест + pre-commit Husky-чек (Hard Rule у AGENTS.md).
- Visual regression: PR-X2 покриває FTUX snapshot-flow.
