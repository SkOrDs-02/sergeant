# Specialists ↔ skills mapping

> **Last validated:** 2026-05-13 by @Skords-01. **Next review:** 2026-08-11.
> **Status:** Active

Цей файл — канонічний міст між **runtime specialist agents** (`SpecialistAgent` у [`tools/console/src/agents/dispatcher.ts`](../../tools/console/src/agents/dispatcher.ts)), які класифікують Telegram-команди (`/assign web-ui …`), і **governance skills** ([`./agent-skills-catalog.md`](./agent-skills-catalog.md)), якими оперують Devin / Claude під час роботи в репо.

> **Чому це потрібно.** До цього мепінгу runtime classification (web-ui, server-api, mobile, …) і governance skill loading (`sergeant-web-ui`, `sergeant-server-api`, `sergeant-mobile-expo`, …) існували паралельно, без явної зв’язки. Якщо Telegram-бот класифікував `/assign web-ui …`, репо-агент мусив вгадувати, який skill завантажити. Тепер mapping однозначний.

> **Marker `extra`** — це специаліст, у якого ще немає 1:1 governance skill. Перед роботою агент має або підтягнути найближчий relevant skill (`sergeant-start-here`), або повідомити, що скоп виходить за межі repo-owned skills. Перетворення `extra` на дедикований skill трекається в initiative 0009 phase 5.1.

## Як це використовується

1. **Telegram dispatcher** (`tools/console/src/agents/dispatcher.ts`) приймає `/assign web-ui …`, класифікує `specialist: "web-ui"`, і в status callback рендерить `Loading skill: sergeant-web-ui` (рендеринг — задача phase 2.1 follow-up; цей PR постачає mapping і `@see` коментарі).
2. **Devin / Claude session** на старті читає `AGENTS.md`, потім `.agents/skills/sergeant-start-here/SKILL.md`, і — якщо в задачі присутній SpecialistAgent-tag (з PR-опису, з webhook payload, або з ручного `/assign`) — завантажує відповідний skill з цієї таблиці.
3. **OpenClaw personas** (`tools/console/src/agents/personas.ts`) — `cofounder` / `eng` / `ops` / `growth` / `finance` — мапляться на skills через primer-text (див. файл): `eng` posses на `sergeant-feature-delivery` і `sergeant-review-and-merge`, `ops` — на `sergeant-deploy-and-observability` тощо. Personas — multi-skill за дизайном (synthesis role); SpecialistAgents — single-skill за convention.

## Як підтримувати свіжість

- При додаванні нового specialist-а в `SpecialistAgent` enum — обовʼязково додати рядок у цю таблицю + JSDoc `@see` посилання в dispatcher.ts.
- При публікації нового skill під `.agents/skills/` — оновити stовпець у відповідному рядку (з `extra` → реальний skill).
- При редагуванні primary playbook'а одного з мепінгів — перевір, чи цей playbook все ще канонічний (часом краще додати другий primary, ніж перевизначати єдиний).
- Лінтер `pnpm lint:skills` сінкає shape кожного SKILL.md, але не валідує цю таблицю безпосередньо. Перевірка drift — manual під час review (за governing skill `sergeant-monorepo-boundaries`).
