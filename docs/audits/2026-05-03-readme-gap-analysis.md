## Частина 2 — Повний гайд по проєкту (простою мовою)

Тут — все, що треба знати, якщо ти перший раз відкрив цей проєкт.

### Що таке Sergeant?

Sergeant — це твій особистий хаб (персональна платформа), яка допомагає керувати:

- **Грошима** (модуль "Фінік") — підключаєш Monobank, бачиш транзакції, встановлюєш бюджети, відстежуєш борги.
- **Тренуваннями** (модуль "Фізрук") — плануєш тренування, записуєш підходи й повторення.
- **Звичками** (модуль "Рутина") — створюєш звички, відмічаєш виконання, бачиш серії (streaks).
- **Харчуванням** (модуль "Харчування") — логуєш прийоми їжі, скануєш штрихкоди продуктів, отримуєш AI-аналіз.
- **Щоденними справами через AI** (HubChat) — спілкуєшся з AI-асистентом (Claude), який може виконувати дії: додати транзакцію, записати тренування, відмітити звичку — все через чат.

### Як це організовано?

Проєкт — це **монорепо** (один великий репозиторій з кількома додатками та пакетами всередині). Інструмент для управління — **Turborepo** + **pnpm** (менеджер пакетів).

#### Додатки (apps/)

| Папка               | Що це                            | Мова                    | Де живе                 |
| ------------------- | -------------------------------- | ----------------------- | ----------------------- |
| `apps/web`          | Веб-сайт (головний інтерфейс)    | React + Vite + Tailwind | Vercel                  |
| `apps/server`       | API-сервер (бекенд)              | Express + PostgreSQL    | Railway                 |
| `apps/mobile`       | Мобільний додаток (нативний)     | Expo + React Native     | App Store / Google Play |
| `apps/mobile-shell` | Мобільний додаток (обгортка web) | Capacitor               | App Store / Google Play |
| `tools/console`     | Telegram-бот (внутрішній)        | grammy + Anthropic      | Railway                 |

#### Пакети (packages/)

Пакети — це шматки коду, які **перевикористовуються** між додатками. Замість копіпастити код з web у mobile, ми кладемо його в `packages/` і імпортуємо звідти.

| Пакет                           | Простими словами                                                                                                             |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `shared`                        | Спільні типи, схеми валідації (Zod), бізнес-логіка. Якщо щось потрібне і вебу, і мобайлу, і серверу — воно тут.              |
| `api-client`                    | Готові функції для виклику API. Замість писати `fetch("/api/transactions")` руками — імпортуєш `finykApi.getTransactions()`. |
| `config`                        | Спільні конфіги ESLint, TypeScript.                                                                                          |
| `design-tokens`                 | Кольори, шрифти, відступи — все, що визначає, як виглядає додаток. Tailwind-пресет теж тут.                                  |
| `insights`                      | Крос-модульна аналітика: чисті функції, які рахують статистику.                                                              |
| `finyk-domain`                  | Математика фінансів: обчислення бюджету, форматування сум (копійки → гривні).                                                |
| `fizruk-domain`                 | Математика фітнесу: калорії, обчислення навантаження.                                                                        |
| `nutrition-domain`              | Математика харчування.                                                                                                       |
| `routine-domain`                | Математика звичок.                                                                                                           |

### Як запустити проєкт локально?

#### Крок 0 — Встанови необхідне

1. **Node.js 20.x** — завантаж з [nodejs.org](https://nodejs.org/) або через Volta:
   ```bash
   curl https://get.volta.sh | bash
   volta install node@20
   ```
2. **pnpm 9.15.1** — менеджер пакетів:
   ```bash
   npm install -g pnpm@9.15.1
   ```
3. **Docker** — для локальної бази даних. Завантаж з [docker.com](https://www.docker.com/).

#### Крок 1 — Склонуй і встанови

```bash
git clone https://github.com/Skords-01/Sergeant.git
cd Sergeant
pnpm install --frozen-lockfile
```

`--frozen-lockfile` означає: "встанови точно ті версії, які записані в lockfile, не чіпай нічого".

#### Крок 2 — Налаштуй змінні середовища

```bash
cp .env.example .env
```

Відкрий `.env` у редакторі. Більшість значень уже заповнені для локальної розробки. Зверни увагу на:

- `DATABASE_URL` — `postgresql://hub:hub@localhost:5432/hub` (дефолт, працює з Docker).
- `BETTER_AUTH_SECRET` — зміни на будь-який рядок з 32+ символів.
- `ANTHROPIC_API_KEY` — якщо хочеш AI-чат, потрібен ключ з [console.anthropic.com](https://console.anthropic.com/).

Решта змінних — опціональні. Кожна має коментар українською, що пояснює, для чого вона.

#### Крок 3 — Підніми базу даних

```bash
pnpm dev:db
```

Ця команда:

1. Запускає PostgreSQL 16 в Docker-контейнері (`docker compose up -d`).
2. Прогоняє всі SQL-міграції (`pnpm db:migrate`).

Якщо треба зупинити базу: `pnpm db:down` (або `docker compose down`).

#### Крок 4 — Запусти сервер і веб

```bash
# В одному терміналі:
pnpm dev:server

# В іншому терміналі:
pnpm dev:web
```

Тепер відкрий:

- **Веб:** [http://localhost:5173](http://localhost:5173)
- **API:** [http://localhost:3000](http://localhost:3000)

#### Крок 5 (опціонально) — Мобільний додаток

```bash
pnpm --filter @sergeant/mobile start
```

Це запустить Expo dev server. Далі відскануй QR-код в Expo Go або запусти на симуляторі.

#### Крок 6 (опціонально) — Telegram-бот

```bash
pnpm --filter @sergeant/console dev
```

Для роботи боту потрібен Telegram Bot Token у `tools/console/.env`.

### Як працює AI-чат (HubChat)?

HubChat — це чат-інтерфейс, де ти спілкуєшся з Claude (AI від Anthropic). Але це не просто чат — Claude може **виконувати дії**:

1. Ти пишеш: "Додай транзакцію -500 грн на каву"
2. Сервер відправляє запит до Anthropic з описом доступних "tools" (інструментів).
3. Claude відповідає: "Я хочу використати tool `create_transaction` з параметрами..."
4. **Клієнт (браузер)** виконує цю дію — записує в localStorage або відправляє на сервер.
5. Результат повертається Claude, і він пише фінальну відповідь.

Важливо: **сервер не виконує дії** — він лише передає запити між тобою та Claude. Всі side effects відбуваються на клієнті.

### Як працює офлайн-режим?

Модулі Фізрук і Рутина працюють за принципом **local-first**:

1. Дані зберігаються спершу в **localStorage** (web) або **MMKV** (mobile).
2. Потім CloudSync синхронізує їх з сервером у фоні.
3. Конфлікти вирішуються за правилом **LWW** (Last-Write-Wins) — перемагає останній запис.

Це означає: навіть без інтернету можна записати тренування або відмітити звичку.

### Як працює час і дати?

Все прив'язане до **часового поясу Київ (Europe/Kyiv)**:

- "Сьогодні" = сьогодні за київським часом (UTC+2 або UTC+3 літом).
- Тиждень починається з понеділка (ISO 8601).
- Не використовуй `new Date().toISOString().slice(0,10)` — це дає UTC-день і ламає серії звичок для користувачів, які працюють пізно ввечері.

### Як працюють гроші?

- У базі й API гроші зберігаються в **мінорних одиницях (копійках)**: 100.50 грн = `10050`.
- Mono API теж повертає копійки — ми зберігаємо як є.
- Ділити на 100 треба **тільки при відображенні** в UI.
- Від'ємне = витрата, додатне = дохід.

### Які є Hard Rules?

Проєкт має **17 жорстких правил** (Hard Rules), які не можна порушувати. Ось стислий огляд:

1. **bigint → number** — PostgreSQL повертає bigint як string, завжди конвертуй через `Number()`.
2. **RQ keys через фабрики** — не хардкодь ключі для React Query.
3. **API-контракт: 3 файли разом** — змінив response → зміни `api-client` → зміни тест.
4. **Міграції послідовні** — без пропусків, два етапи для DROP.
5. **Conventional Commits** — тільки дозволені типи та scope.
6. **Заборона force push** — в `main`/`master`.
7. **Husky хуки** — не вимикай `--no-verify`.
8. **Opacity кроки** — тільки з зареєстрованої шкали.
9. **`-strong` для тексту на фоні** — для WCAG AA 4.5:1.
10. **Lifecycle маркери** — `@scaffolded`, `@deprecated`, `@experimental`.
11. **Заборона hex-кольорів** — у Tailwind className.
12. **Один акцент = один модуль** — не міксуй кольори модулів.
13. **Заборона сирих dark: пар** — використовуй семантичні токени.
14. **`focus-visible:` замість `focus:`** — для кільця фокусу.
15. **Читай governance, оновлюй docs, пиши українською** — перед кодингом.
16. **Типографічна шкала** — мін. 12px, семантичні стилі.
17. **Бюджет анімацій** — макс. 2 одночасно, 3 тіри.

Детально: `AGENTS.md` → секція "Hard rules".

### Типові проблеми та рішення

| Симптом                                  | Причина                                            | Як полагодити                                                   |
| ---------------------------------------- | -------------------------------------------------- | --------------------------------------------------------------- |
| `pnpm install` падає з помилкою lockfile | Неправильна версія pnpm                            | `npm i -g pnpm@9.15.1`                                          |
| `pnpm dev:db` не працює                  | Docker не запущений                                | Запусти Docker Desktop, потім повтори                           |
| Порт 5432 зайнятий                       | Інший Postgres або контейнер                       | `docker ps` → зупини конфліктний контейнер                      |
| API каже "401 Unauthorized"              | Немає сесії або `BETTER_AUTH_SECRET` не збігається | Перезапусти сервер після зміни `.env`, зареєструйся знову       |
| HubChat каже "Невідома дія"              | `max_tokens` обрізав JSON tool-call                | Не зменшуй `max_tokens` без тестування                          |
| lint-staged падає на commit              | Код не пройшов ESLint / Prettier                   | Виправ помилки, `pnpm lint --fix`                               |
| `text-3xs` не рендериться                | Токен видалений (Hard Rule #16)                    | Заміни на `text-xs` або `text-2xs` (лише для дрібних декорацій) |
| Серії (streaks) збиваються               | Використаний UTC замість Kyiv                      | Завжди рахуй "сьогодні" через `Europe/Kyiv`                     |
| Числа з API приходять як string          | bigint → string (PostgreSQL pg driver)             | `Number(r.id)` у серіалізаторі                                  |
